    # (static) cloneEl(el, optionsopt, cloneCountopt) → {HTMLElement|Array.<HTMLElement>}

    This method clones an element or an array of elements.
    It gives you the option of deep cloning(i.e clone children) or just the element itself.
    By default, deepClone is used on the element, which means both the elements children are cloned.

    Parameters:
    Name Type Attributes Default Description
    el String | Array.<String>

    A valid css selector string or an array of strings and elements representing an HTML element
    options Object | ToolJSNodeListManipulator <optional>

    A key to pair object that specifies how the element(s) are cloned.
    If passed a boolean, then its sts the deepClone parameter and if passed a number, its sets the clone count parameter

    Properties
    Name Type Attributes Default Description
    deepClone Object <optional>
    		 true

    Specifies whether to clone both the elements children
    count Number <optional>
    		 1

    Specifies the number of times the clone is made.
    cloneCount Number <optional>
    		 1

    This also optionally specifies the number of times the clone is made.
    Source:
    • tooljs.esm.js, line 2944
    Returns:

    an html element or an array of elements

    Type
    HTMLElement | Array.<HTMLElement>
    Example 
    // returns a single clone of each element in the array
var myClones = DOM.cloneEl(["h4", "p", "#myDiv1"]);

// returns a single clone of the "div" element without its children
var myClone = DOM.cloneEl("#myDiv1", false, 1);

// returns 2 clones of the "div" element without its children and appends them to "#newDiv"
var myClone = DOM.cloneEl("#myDiv1", {
    deepClone: false,
    count: 2,
    appendTo: "#newDiv"
});

    # ToolJSNodeListManipulator

    This defines a of key to pair object which manipulates DOM elements in a ToolJSNodeList instance.
    This options help to do a quick manipulation on the DOM element(s). The options object is usually the last parameter of the methods in the DOM module.
    All methods in the DOM module have this parameter except dataAttr(), each(), on(), off(), isNodeList(), isCollection(), removeChild(), removeEl(), offset() and isEl()
    It comes with a set of pre-defined properties, while the rest are the same properties every DOM element has in your native javascript such as "id", "className", "title", "tabIndex" etc.

    Below are a list of the pre-defined properties available

    Type:
    • Object
    Properties
    Name Type Attributes Description
    insertBefore Object <optional>

    This options moves the element using the insertBefore method.
    insertAfter Object <optional>

    This options moves the element using the insertAfter method.
    appendTo Object <optional>

    This options moves the element using the appendTo method.
    css String <optional>

    Adds a css style string to an element. E.g "color: red; font-size: 40px".
    style Object <optional>

    Accepts an object of javascript style properties and values. E.g {color: "red", fontSize: "40px"}.
    children Object <optional>

    Manipulates the children of the reference element.
    It takes any option in the ToolJSNodeListManipulator object. It basically replicates the .childrenEl method.
    siblings Object <optional>

    Manipulates the siblings of the reference element, replicating the .siblings method and also providing two(2) options.
    It accepts an objcet itself as a value, and if passed a valid css selector string as key of one of its object properties, then a matching element is looked for and returned for manipulation.
