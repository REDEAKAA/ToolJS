    Tutorial: Getting Started

    Getting Started

    Using ToolJS

    The library was designed to work in most popular JavaScript environment such as a CommonJS environment(e.g NodeJs) or in the Browser. With this in mind the libraries repo provides you with three download options.

    This section assumes you have downloaded or included one of the files made available

    The ToolJS class

    The library exports a class ToolJS as its default export and several named exports(modules) for the purpose of tree-shaking, so you can include only the modules you need in your project

    Configuring your ToolJS work environment

    The ToolJS class provides a method .config() for you to configure your ToolJS instance. It accepts a key to value object consisting of configuration options. See ToolJSConfig for all available config options 

        ToolJS.config({
        debugging: true // allow debugging and error logs
        ...
    });

    Exporting Methods

    Now that you have configured your ToolJS environment, its time to export some methods which we can use in our project into a variable. This can be done using the .export() method on the ToolJS class. Pass it a string representing a registered module or an array or strings, and it will export every available method in each of the modules.
    If no paramter is passed to it, then it exports all methods in all registered module 

        // this exports all methods in all registered module into the "$" variable
    var $ = ToolJS.export();

    // this exports all methods in the DOM module into the "MyDOM" variable
    var MyDOM = ToolJS.export("DOM");

    // this exports all methods in the DOM and Str modules into the "Methods" variable
    var Methods = ToolJS.export(["DOM", "Str"]);

    Using Exported Methods

    After exporting methods from modules, you can use them as methods of the variable to which they were exported to 

        // this exports all methods in all registered module into the "$" variable
    var $ = ToolJS.export();

    // gets the element in the DOM with an id of "para1"
    var myP = $.getEl("#para1");

    // style the element using the style method
    $.styleEl(myP, {
        color: "red",
        fontFamily: "Comic Sans MS"
    });

    Writing a Plugin(Module)

    Because the library is open-source and extensible, you can write plugins for the library and upload online. This plugins are regarded as modules which can be added to an instance of the library. See Create a Plugin
